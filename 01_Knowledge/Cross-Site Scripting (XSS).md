The browser's security model is built on the **Same-Origin Policy (SOP)**, which ensures that code downloaded from different websites runs in complete isolation. A **Genuine Script** running on **Site A (Genuine)** is strictly blocked from accessing data, objects, or credentials belonging to **Site B (Attacker)**.
![[same-origin.svg]]

In a successful **Cross-Site Scripting (XSS)** attack, <mark style="background: #FFB8EBA6;">the attacker bypasses this isolation boundary. By forcing an **Attack Payload** from **Site B (Attacker)** into the data stream of **Site A (Genuine)**,</mark> they trick the browser into running an **Attack Script** inside the secure runtime environment of **Site A (Genuine)**.
![[xss.svg]]
Because the browser now mistakes this **Attack Script** for a **Genuine Script** belonging to **Site A**, the malicious code gains full clearance to do anything the legitimate application code can do—including reading memory, stealing session tokens, or making authorized API requests on behalf of the user.
